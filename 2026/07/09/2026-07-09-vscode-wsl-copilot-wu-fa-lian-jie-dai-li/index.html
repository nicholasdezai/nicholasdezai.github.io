如果 VS Code 通过 WSL 使用 Copilot，而代理始终不起作用，可以在用户的 settings.json 中添加：

这样会强制 Copilot 扩展运行在本地（UI）而不是 WSL（Workspace）中，从而直接使用本机代理。

简单来说：

  • ui：扩展运行在本地 Windows/macOS/Linux。
  • workspace：扩展运行在远程环境（WSL、SSH、容器）。

对于需要走本机代理的扩展，通常设置为 ui 即可。
