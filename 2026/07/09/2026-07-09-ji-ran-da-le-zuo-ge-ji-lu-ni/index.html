配置评论系统 Giscus

Butterfly 支持很多评论系统。

排名第一的 Waline 我看了一下需要自己部署服务，而我比较懒（笑），最后选择了 Giscus。

Giscus 基于 GitHub Discussions，因此不需要自己维护数据库，非常适合个人博客。

它的大致工作流程如下：

Giscus 会根据页面 URL、标题等信息，在 GitHub Discussions 中查找对应的话题。如果没有找到，在第一次有人评论时，会自动创建对应的 Discussion。
评论用户需要使用 GitHub 登录，因此垃圾评论也比较少。

配置步骤：

  1. 新建一个 GitHub Repository。
  2. 在 Settings → General 中开启：
    • Issues
    • Discussions
  3. 打开 Giscus 官网：
    • https://giscus.app/zh-CN
  4. 根据页面提示生成配置。
  5. 将生成的配置填入 Butterfly 的评论配置中即可。

分类（Category）我选择的是 General，其实选哪个都可以。

Butterfly 官方文档也写得很详细：

VS Code WSL 中 Copilot 无法连接代理
如果 VS Code 通过 WSL 使用 Copilot，而代理始终不起作用，可以在用户的 settings.json 中添加： 1234"remote.extensionKind": { "GitHub.copilot": ["ui"], "GitHub.copilot-chat": ["ui"]} 这样会强制 Copilot 扩展运行在本地（UI）而不是 WSL（Workspace）中，从而直接使用本机代理。 简单来说： ui：扩展运行在本地 Windows/macOS/Linux。 workspace：扩展运行在远程环境（WSL、SSH、容器）。 对于需要走本机代理的扩展，通常设置为 ui 即可。
